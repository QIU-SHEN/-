# 🤖 RPA 模式使用指南

使用浏览器自动化来发布文章到微信公众号，**支持个人未认证公众号**。

## 📋 原理说明

RPA (Robotic Process Automation) 通过模拟真人操作浏览器来完成发布：

```
AI生成文章 → 打开浏览器 → 访问公众号后台 → 扫码登录 → 新建图文 → 
填写内容 → 上传封面 → 保存/发布
```

## ✅ 优势 vs 局限

| | API 模式 | RPA 模式 |
|---|---------|---------|
| 公众号要求 | 需要认证号 | ✅ 支持个人号 |
| 稳定性 | 高 | 中等（依赖网页结构） |
| 速度 | 快 | 较慢（模拟人工操作） |
| 需要扫码 | 否 | ✅ 首次需要 |
| 微信限制 | 群发次数限制 | 无限制 |

## 🚀 快速开始

### 1. 安装依赖

```bash
cd D:\OpenClaw-Projects\wechat-ai-assistant
pip install playwright pyperclip
playwright install chromium
```

### 2. 配置启用 RPA

编辑 `config/config.yaml`：

```yaml
wechat:
  # 启用 RPA 模式
  use_rpa: true
  rpa_headless: false  # false = 显示浏览器（首次需要扫码）
  
  publish:
    auto_publish: false  # 建议先 false，保存草稿人工确认
```

### 3. 首次运行（扫码登录）

```bash
python src/main.py --workflow create --debug
```

首次运行会：
1. 打开浏览器窗口
2. 访问微信公众号后台
3. 显示二维码
4. **你需要用微信扫码登录**
5. 登录状态会自动保存

### 4. 后续运行（自动登录）

保存登录状态后，后续运行会自动登录，无需再次扫码。

## 📱 工作流程

### 手动发布流程（推荐）

```
1. AI 生成文章
2. 浏览器自动打开公众号后台
3. 自动填写标题、作者、正文
4. 自动上传封面图片
5. ✅ 保存为草稿
6. 人工在后台点击"群发"
```

### 全自动发布

```yaml
wechat:
  publish:
    auto_publish: true  # 开启自动发布
```

⚠️ **警告**：开启后 AI 生成的内容会直接发布，请确保内容质量！

## 🔧 高级配置

### 无头模式（服务器运行）

```yaml
wechat:
  rpa_headless: true  # 不显示浏览器窗口
```

**注意**：首次必须在有图形界面的环境下运行，完成扫码登录后再切换到无头模式。

### 登录状态保存

登录状态默认保存在 `./data/rpa_state/wechat_login_state.json`

如果需要重新登录，删除该文件即可。

## ⚠️ 常见问题

### Q: 扫码后无法自动跳转？
A: 检查网络连接，或增加超时时间。

### Q: 正文内容格式错乱？
A: 公众号编辑器对格式要求较严，建议使用纯文本。

### Q: 登录状态过期？
A: 删除 `data/rpa_state/wechat_login_state.json` 后重新扫码。

## 🛡️ 安全建议

1. **不要在公共服务器上保存登录状态**
2. **定期清理登录状态文件**
3. **频繁操作可能触发微信风控**

---

**注意**：RPA 方式依赖于微信公众号网页版的结构，如果微信更新页面布局，可能需要更新代码。
